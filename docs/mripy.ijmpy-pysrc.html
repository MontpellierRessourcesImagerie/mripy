  1  '''  
  2  ijmpy is a python wrapper of the imagej-macro-language 
  3   
  4  It allows to move ij-macro code to python (mostly) without modifications (yes, you can keep the ';') 
  5  The aim is to facilitate the step of moving from macro to scripting. You can record your  
  6  macro, use the familiar macro language and learn how to profit from the better code organization 
  7  and data-structures (list, tupels, dictionary, ...) in python. 
  8   
  9  written 2020 by `Volker Baecker`_, `Montpellier Ressources Imagerie`_, Biocampus Montpellier, UM, INSERM, CNRS 
 10   
 11  .. _`Volker Baecker`: https://github.com/volker-baecker 
 12  .. _`Montpellier Ressources Imagerie`: http://www.mri.cnrs.fr 
 13   
 14  Have fun!

 18   
 19  abs() 
 20  ----- 
 21   
 22  When you call 'abs()' the python 'abs()' function is called. The behaviour should be identical. 
 23   
 24  acos() 
 25  ------ 
 26   
 27  Import the function from python's math module instead.  
 28   
 29  .. code-block:: python 
 30   
 31          from math import acos 
 32   
 33  Other differences with the ij-macro language 
 34  ============================================ 
 35   
 36  output parameter 
 37  ---------------- 
 38   
 39  Functions that take output parameter in the macro language are implemented without these parameters  
 40  and return the results as an n-tupel instead. 
 41   
 42  For example, use: 
 43   
 44          min, max, mean, stddev = Array.getStatistics(anArray) 
 45   
 46  instead of 
 47   
 48          Array.getStatistics(anArray, min, max, mean, stddev) 
 49   
 50  Only in the case that the parameter modified in the function is an array, it is modified in place. 
 51  For example: 
 52   
 53          Array.fill(array, 0) 
 54          Array.reverse(array)
